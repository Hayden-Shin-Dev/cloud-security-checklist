# Cloud Security Checklist

Cloud Security Checklist는 클라우드 환경의 핵심 보안 통제 항목을 빠르게 점검하고, 보안 점수와 위험 등급, 카테고리별 취약 영역, 개선 권고사항을 확인할 수 있는 Streamlit 기반 대시보드입니다.

이 프로젝트는 별도 데이터베이스 없이 로컬에서 실행되며, Azure, AWS, GCP, On-Premise 환경의 기본 보안 상태를 간단하고 일관된 방식으로 평가하는 데 초점을 둡니다.

## 주요 기능

- 클라우드 환경 선택: Azure, AWS, GCP, On-Premise
- 회사명 또는 프로젝트명 입력
- 15개 핵심 보안 체크리스트 항목 제공
- 항목별 가중치 기반 100점 만점 보안 점수 계산
- 위험 등급 자동 산정: 양호, 주의, 위험, 매우 위험
- 카테고리별 점수 표시
- 미충족 항목에 대한 구체적인 개선 권고사항 제공
- 전체 선택 및 전체 초기화 기능
- 현재 날짜와 선택한 클라우드 환경이 포함된 CSV 다운로드
- 모바일 화면에서도 사용 가능한 반응형 Streamlit 레이아웃

## 설치 방법

Python 3.11 이상이 설치되어 있는 환경에서 다음 명령을 실행합니다.

```bash
git clone https://github.com/Hayden-Shin-Dev/cloud-security-checklist.git
cd cloud-security-checklist
python -m venv .venv
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

macOS 또는 Linux:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

## 실행 방법

```bash
streamlit run app.py
```

실행 후 브라우저에서 Streamlit이 안내하는 로컬 주소로 접속합니다.

일반적으로 다음 주소 중 하나로 접속할 수 있습니다.

```text
http://localhost:8501
http://127.0.0.1:8501
```

## 보안 점수 계산 방식

각 체크리스트 항목은 중요도에 따라 가중치를 가지며, 전체 가중치 합계는 100점입니다.

사용자가 체크한 항목의 가중치를 합산하여 최종 보안 점수를 계산합니다.

```text
보안 점수 = 체크된 항목의 가중치 합계 / 전체 가중치 합계 * 100
```

현재 전체 가중치 합계는 100이므로, 체크된 항목의 가중치 합계가 곧 최종 점수입니다.

### 위험 등급 기준

| 점수 구간 | 등급 | 의미 |
| --- | --- | --- |
| 80점 이상 | 양호 | 핵심 보안 통제가 대체로 적용된 상태입니다. |
| 60점 이상 80점 미만 | 주의 | 주요 통제는 일부 적용되어 있으나 개선이 필요합니다. |
| 40점 이상 60점 미만 | 위험 | 다수의 핵심 통제가 미흡하여 우선 조치가 필요합니다. |
| 40점 미만 | 매우 위험 | 기본 보안 통제 부재 가능성이 높아 즉시 개선이 필요합니다. |

### 카테고리 구성

| 카테고리 | 주요 점검 영역 |
| --- | --- |
| 계정 및 접근 관리 | MFA, 개인별 계정, 최소 권한 |
| 네트워크 보안 | Default Deny, SSH/RDP 공개 제한, DB 직접 노출 방지 |
| 데이터 보호 | 저장 데이터 암호화, 전송 구간 암호화 |
| 로그 및 모니터링 | 시스템/접근 로그 수집, 비정상 접근 알림 |
| 백업 및 운영 관리 | 정기 백업, 복구 테스트, 환경 분리, 변경 승인 |

## 테스트해야 할 항목

실행 후 다음 항목을 확인합니다.

1. 사이드바에서 클라우드 환경을 Azure, AWS, GCP, On-Premise로 변경할 수 있는지 확인합니다.
2. 회사명 또는 프로젝트명을 입력했을 때 CSV 결과에 반영되는지 확인합니다.
3. 모든 항목 선택 버튼을 눌렀을 때 총 보안 점수가 100점, 위험 등급이 양호로 표시되는지 확인합니다.
4. 모든 선택 초기화 버튼을 눌렀을 때 총 보안 점수가 0점, 위험 등급이 매우 위험으로 표시되는지 확인합니다.
5. 일부 항목만 선택했을 때 카테고리별 점수와 총점이 가중치 기준으로 계산되는지 확인합니다.
6. 체크하지 않은 항목이 주요 위험 요소로 표시되고, 각 항목별 개선 권고사항이 보이는지 확인합니다.
7. CSV 다운로드 파일에 점검일, 회사명 또는 프로젝트명, 클라우드 환경, 점검 항목, 적용 여부, 가중치, 총점, 위험 등급이 포함되는지 확인합니다.
8. 모바일 폭 또는 좁은 브라우저 창에서도 주요 지표와 체크리스트가 크게 깨지지 않는지 확인합니다.

## 운영 및 활용 예시

이 대시보드는 정식 보안 진단 도구를 대체하기보다는, 초기 보안 현황 파악과 개선 우선순위 정리에 적합합니다.

- 신규 클라우드 프로젝트 착수 전 기본 보안 통제 점검
